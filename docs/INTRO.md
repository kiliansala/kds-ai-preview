# Kapsch Design System

> AI-generated Design System PoC — Figma as Single Source of Truth, components validated end-to-end across React, Angular, and Blazor.

## What is this?

KDS AI Preview demonstrates how to build an automated Design System where **Figma is the single source of truth**. Every component is extracted from Figma via MCP, validated against a design contract, and shipped as a framework-agnostic web component with wrappers for React, Angular, and Blazor.

All code is generated by Claude AI following a repeatable 9-step workflow per component.

## Key Pillars

| Pillar | Description |
|--------|-------------|
| **Figma SSOT** | Component properties and tokens extracted directly from Figma via MCP |
| **Contract Validation** | Automated check that code implements 100% of the Figma design |
| **DTCG Tokens** | 90+ design tokens in the Design Tokens Community Group standard format |
| **LIT Web Components** | Framework-agnostic components using LIT 3.x and Shadow DOM |
| **Framework Wrappers** | Idiomatic APIs for React, Angular 20/21, and Blazor |
| **WCAG 2.1 AA** | Every component passes automated accessibility validation |

## Quick Start

```bash
# Install dependencies
npm install

# Dev mode — interactive documentation
npm run dev --workspace=@kds/web-components
# Opens http://localhost:5173

# Build all packages
npm run build --workspaces
```

## Using a Component

```html
<!-- Web Component -->
<script type="module" src="@kds/web-components"></script>
<kds-button size="md" hierarchy="primary">Click me</kds-button>
```

```tsx
// React
import { Button } from '@kds/react';
<Button size="md" hierarchy="primary" onClick={handleClick}>Click me</Button>
```

```typescript
// Angular
import { KdsButtonComponent } from '@kds/angular';
```

```razor
@* Blazor *@
<KdsButton Size="md" Hierarchy="primary" OnClick="@HandleClick">Click me</KdsButton>
```
